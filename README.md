# AIGC Reducer - 学术论文AIGC降重工具

基于规则引擎的学术论文AIGC检测率降低工具。核心思路：**打破AI文本的可预测性**，而不是简单的同义词替换。

## 安装

```bash
pip install -r requirements.txt
```

## 快速开始

### 1. 全文扫描模式（无需检测报告）

自动扫描全文中的AI写作特征，并进行修改：

```bash
python reducer.py --doc 你的论文.docx
```

### 2. 精准模式（推荐，需要检测报告）

先用AIGC检测工具跑一次，拿到PDF报告后精准修改：

```bash
python reducer.py --doc 你的论文.docx --report 检测报告.pdf
```

### 3. 仅分析模式

先看看论文有哪些AI特征，不做任何修改：

```bash
python reducer.py --doc 你的论文.docx --analyze-only
```

### 4. 交互模式

逐段确认是否应用修改，适合对论文质量要求高的场景：

```bash
python reducer.py --doc 你的论文.docx --interactive
```

### 5. 导出/导入替换映射

先导出为JSON方便手动编辑，确认后再导入应用：

```bash
# 导出
python reducer.py --doc 你的论文.docx --export-json replacements.json

# 手动编辑 replacements.json 后导入
python reducer.py --doc 你的论文.docx --import-json replacements.json
```

## 降重力度

通过 `--level` 参数控制（默认2）：

| 等级 | 说明 | 适用场景 |
|------|------|----------|
| 1 | 轻微 — 仅替换序列连接词和学术套话 | 检测率略高，只需微调 |
| 2 | 中等 — 打破句式对称 + 注入主观标记 | 大多数场景推荐 |
| 3 | 激进 — 拆句 + 主被动转换 + 全面重构 | 检测率顽固居高不下 |

## 完整参数列表

```
--doc           输入的docx论文文件路径（必需）
--report        AIGC检测报告PDF路径
--output, -o    输出文件路径（默认: 原文件名_降重版.docx）
--level         降重力度: 1/2/3
--analyze-only  仅分析AI特征，不修改
--interactive   交互式逐段确认
--export-json   导出替换映射到JSON文件
--import-json   从JSON文件导入替换映射
--skip-headings 跳过标题段落（默认开启）
--skip-english  跳过英文段落（默认开启）
--min-length    忽略字数少于此值的段落（默认20）
```

## 降重策略原理

本工具基于以下实战总结的AI检测规避策略：

### 1. 打破序列词模式
AI偏好使用"首先…其次…此外…综上所述"这种结构。工具会将其替换为因果链式表达。

### 2. 破坏句式对称性
AI生成的文本倾向于使用并列、对称的长句。工具会拆分过长的并列句，打断节奏。

### 3. 注入人类写作痕迹
插入"笔者认为""不可否认的是""坦白讲"等主观标记，降低文本的"平滑度"指标。

### 4. 替换通用连接词
将"因此""然而""同时"等高频AI用词替换为更有变化的表达。

### 5. 重构句子结构（激进模式）
拆分长句、转换主被动语态，从结构层面改变文本特征。

## 工作流建议

```
第一次检测 → 分析模式看风险分布 → 中等力度降重 → 再次检测
    ↓                                              ↓
如果>20%                                      如果<10% → 完成
    ↓                                              
激进模式 + 手动修改高风险段 → 再次检测
```

## 手动优化技巧（工具做不到的部分）

1. **注入具体细节**：在高风险段落中加入具体法条编号、学者姓名、政策文件日期
2. **场景化叙述**：把抽象论述改为"设想一个场景…"的叙事方式
3. **段落重组**：将红色标记的段落拆分后与相邻安全段落合并
4. **个人观点**：适当加入"笔者在调研中发现…""从实务角度看…"

## 文件结构

```

└── README.md           # 本文件
```

## 注意事项

- 工具基于规则引擎，效果取决于论文的AI特征强度
- 对于顽固的高风险段落，建议手动重写（效果远优于规则替换）
- 降重后务必通读全文，确保学术语义和逻辑未被破坏
- 英文段落默认跳过（通常不参与中文AIGC检测）
