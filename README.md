# AIGC Reducer - 学术论文AIGC降重工具

AIGC Reducer 是一个面向学术论文场景的 AIGC 检测率优化工具，支持规则降重、AI 深度降重、检测报告驱动的混合策略，以及术语保护与可视化预览。核心思路不是机械同义词替换，而是**打破 AI 文本的高可预测性，同时保留原始学术语义与关键术语**。

## 安装

```bash
pip install -r requirements.txt
```

## 核心功能

### 1. API 动态配置
Web 端支持动态配置 OpenAI 兼容接口，无需写死环境变量。可配置内容包括：

- API Base URL
- API Key
- Model Name
- Temperature
- 额外 AI 指令（Custom Prompt）

支持 OpenAI、DeepSeek、通义千问以及其他兼容 `/v1/chat/completions` 的平台。

### 2. 术语保护库
支持在降重前将用户指定术语替换为占位符（如 `[KEYWORD_0]`、`[KEYWORD_1]`），在改写完成后自动恢复，避免模型误改：

- 专业术语
- 变量名
- 法条名称
- 模型名
- 固定短语

该机制仅作用于文本层，不直接改动 `.docx` 的段落格式结构。

### 3. 混合降重策略（基于检测报告）
当用户上传 AIGC 检测报告并启用混合模式后，系统会根据段落风险分层处理：

- 高风险 / 中风险段落：使用 AI 深度改写
- 低风险段落：使用规则引擎轻量处理
- 无风险段落：跳过，不做改写

这可以在控制成本的同时，把 AI 资源集中用在真正高风险的段落上。

### 4. 可视化预览与方法透明化
前端 Compare View 会对每个已处理段落展示处理方式标签：

- `[AI Mode]`
- `[Rule Mode]`
- `[Original/Skipped]`

方便快速确认每个段落到底是通过 AI、规则，还是原样保留。

## 快速开始

### 1. 全文扫描模式（无需检测报告）

自动扫描全文中的 AI 写作特征，并进行修改：

```bash
python reducer.py --doc 你的论文.docx
```

### 2. 精准模式（推荐，需要检测报告）

先使用 AIGC 检测工具生成 PDF 报告，再进行针对性降重：

```bash
python reducer.py --doc 你的论文.docx --report 检测报告.pdf
```

### 3. 仅分析模式

只分析论文中的 AI 写作特征，不执行修改：

```bash
python reducer.py --doc 你的论文.docx --analyze-only
```

### 4. 交互模式

逐段确认是否应用修改，适合对成稿质量要求较高的场景：

```bash
python reducer.py --doc 你的论文.docx --interactive
```

### 5. 导出 / 导入替换映射

先导出 JSON 进行人工审校，确认后再导入：

```bash
# 导出
python reducer.py --doc 你的论文.docx --export-json replacements.json

# 编辑 replacements.json 后导入
python reducer.py --doc 你的论文.docx --import-json replacements.json
```

## Web 版使用说明（V1.1+）

### AI 配置
在 Web 界面的 AI 配置面板中，可填写：

- API Base URL
- API Key
- Model Name（默认 `gpt-3.5-turbo`）
- Temperature
- 额外 AI 指令

点击“测试连接”后，后端会请求 `/api/test_llm` 验证 Key 与模型接口是否可用。配置会自动保存到浏览器 LocalStorage，并在下次打开页面时自动回填。

### 高级降重设置
在“高级降重设置”卡片中，可填写或切换：

- 术语保护库（多行或逗号分隔）
- 混合降重开关（仅对高风险段落使用 AI）

### 推荐工作流

```text
上传论文 → 上传检测报告（可选但推荐）→ 分析风险 → 配置 AI 与术语保护 →
启用混合模式 → 执行降重 → Compare View 检查每段处理方式 → 下载文档
```

## 降重力度

通过 `--level` 参数控制（默认 2）：

| 等级 | 说明 | 适用场景 |
|------|------|----------|
| 1 | 轻微 — 仅替换序列连接词和学术套话 | 检测率略高，只需微调 |
| 2 | 中等 — 打破句式对称 + 注入主观标记 | 大多数场景推荐 |
| 3 | 激进 — 拆句 + 主被动转换 + 全面重构 | 检测率顽固居高不下 |

---

## CLI 参数说明

```text
--doc           输入的 docx 论文文件路径（必需）
--report        AIGC 检测报告 PDF 路径
--output, -o    输出文件路径（默认: 原文件名_降重版.docx）
--level         降重力度: 1/2/3
--analyze-only  仅分析 AI 特征，不修改
--interactive   交互式逐段确认
--export-json   导出替换映射到 JSON 文件
--import-json   从 JSON 文件导入替换映射
--skip-headings 跳过标题段落（默认开启）
--skip-english  跳过英文段落（默认开启）
--min-length    忽略字数少于此值的段落（默认 20）
```

> 说明：术语保护库、混合降重开关、额外 AI 指令、动态 API 配置目前主要在 Web 版中提供完整交互能力。

## 降重策略原理

本工具基于以下实战总结的 AI 检测规避策略：

### 1. 打破序列词模式
AI 偏好使用“首先…其次…此外…综上所述”这种结构。工具会将其替换为因果链式表达。

### 2. 破坏句式对称性
AI 文本容易出现长句、并列结构和节奏过于规整的问题。工具会拆分长句并打断对称表达。

### 3. 注入人类写作痕迹
插入“笔者认为”“不可否认的是”“坦白讲”等主观标记，降低文本的平滑度与模板化特征。

### 4. 替换通用连接词
将“因此”“然而”“同时”等高频 AI 用词替换为更自然、更具变化的表达。

### 5. 占位符术语保护
对于关键术语，先替换为占位符再执行改写，结束后恢复原词，降低专业名词被误改的风险。

### 6. 风险分层混合处理
结合检测报告时，高中风险段落走 AI，低风险段落走规则，无风险段落跳过，从而兼顾效果、成本与可控性。

## 工作流建议

```text
第一次检测 → 分析模式看风险分布 → 中等力度降重 → 再次检测
    ↓                                              ↓
如果 >20%                                     如果 <10% → 完成
    ↓
激进模式 / 混合模式 + 手动修改高风险段 → 再次检测
```

## 手动优化技巧（工具做不到的部分）

1. **注入具体细节**：在高风险段落中加入法条编号、学者姓名、政策日期、实验参数等具体信息
2. **场景化叙述**：把抽象论述改成具体问题情境或案例化表达
3. **段落重组**：将高风险段拆分后与相邻安全段合并，打散 AI 模板痕迹
4. **个人观点**：适当加入“笔者在调研中发现…”“从实务角度看…”等表达

## 注意事项

- 工具基于规则与提示词工程结合的策略，效果取决于原文风险分布与报告质量
- 对于顽固高风险段落，手动重写仍然是效果最强的方式
- 降重后务必通读全文，确保学术语义、逻辑链路和专业术语未被破坏
- 英文段落默认跳过（通常不参与中文 AIGC 检测）
- 混合模式依赖检测报告与可用的 LLM API；若未上传报告，则自动回退到纯规则模式
